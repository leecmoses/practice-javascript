# Objects in Depth

1. Introduction

  • The array is one of the most useful data structures in JavaScript, at its core, an array is just an ordered collection of elements, enclosed by square brackets
  • Each element in an array is referenced by a numeric key called an index, which starts from zero and increments by one for each additional element in the array
  • Objects is another very important data structure in JavaScript, fundamentally, it is a collection of associated key/value pairs with curly brackets around it
  • Unlike arrays, objects are unordered collections
  • For object key names you'll commonly find quotation marks omited from property names, however certain situations require them to be included, especially if the property name:
    • Is a reserved word (e.g., for, if, let, true, etc.)
    • Contains spaces or special characters that cannot appear in a variable name (i.e., punctuation other that $, and _ -- most accented characters)
  • There are two ways to retrieve information from objects: dot notation and square bracket notation
  • While dot notation may be easier to read and write, it can't be used in every situation, for example if a key is a number or when variables are assigned to property names

2. Create and Modify Properties

  • To create a new, blank object, you can use object literal notation, or the Object() constructor function
  • While both methods ultimately return an object without properties of its own, the Object() constructor function is a bit slower and more verbose
    • As such, the recommended way to create new objects in JavaScript is to use literal notation
  • Since objects are passed by reference, making changes to the copy has a direct effect on the original object as well

Invoking Object Methods

  • A method is a property of an object whose value is a function
  • Using this, methods can directly access the object that it is called on
  • Note that this is a reserved word in JavaScript, and cannot be used as an identifier (e.g. variable names, function names, etc.)
  • The value of this is set when a method is invoked on an object, and that value refers to that object
  • Arrow function expressions are special: they don't have their "own" this, instead it takes from the outer "normal" function

4. Beware of Globals

  • How the function is invoked determines the value of this inside the function
  • The window object is provided by the browser and is not part of the JavaScript language or specification
  • Since the window object is at the highest (i.e., global) level, an interesting thing happens with global variable declarations
    • Every variable declaration that is made at the global level (outside of a function) automatically becomes a property on the window object
    • Only declaring variables with the var keyword will add them to the window object
    • If you declare a variable outside of a function with either let or const, it will not be added as a property to the window object
  • Any global function declarations are accessible on the window object as methods
  • Avoid global variables and functions for two reasons, tight coupling and name collisions
  • Tight coupling is a phrase that developers use to indicate code that is too dependent on the details of each other
  • Name collision occurs when two (or more) functions depend on a variable with the same name
    • A major problem wit this is that both functions will try to update the variable and or set the variable, but these changes are overridden by each other
  • There are times when you'll need to write global variables, but you should only write them as a last resort

5. Extracting Properties and Values

  • Object.keys() returns an array of the provided object's property names in the same order as the elements would be when using a for loop
  • Likewise, if we want a list of the values of an object, we can use Object.values()

2. First-Class Functions

  • In JavaScript, functions are first-class functions, this means that you can do with a function just about anything you can do with other elements, such as numbers, strings, objects, arrays, etc
  • JavaScript functions can:
    • Be stored in variables
    • Be returned from a function
    • Be passed as arguments into another function
  • In many ways, a function in JavaScript can be treated as a value
  • A function that returns another function is know as higher-order function
  • First-class function

3. Callbacks

  • A function passed into another function as an argument is called a callback function
  • Array methods, such as forEach(), map(), and filter(), take advantage of callbacks to execute functions onto a given array's elements
